# Locus - Анализатор контекста кода для LLM

**Locus** — это гибкий инструмент для анализа структуры проекта и экспорта контекста кодовой базы в формате, удобном для LLM. Помогает выбрать и подготовить нужный контекст кода для работы с AI-чатами.

[🇬🇧 English version](README.md)

## 🎯 Назначение

При работе с LLM (ChatGPT, Claude и др.) нужно предоставлять релевантный контекст кода. Locus помогает:

- 📊 **Анализировать** структуру проекта и визуализировать её в виде дерева
- 📦 **Экспортировать** код модульными группами, оптимизированными для контекстных окон LLM
- 🎨 **Форматировать** вывод в виде Markdown-отчётов или текстовых файлов
- 🔍 **Фильтровать** файлы по паттернам и глубине, фокусируясь на важном
- 📝 **Извлекать** аннотации, docstring'и и сигнатуры функций
- 🎛️ **Настраивать** пользовательские правила группировки через `.locus/settings.json`

## 🚀 Быстрый старт

### Установка

```bash
pip install -e .
```

### Базовое использование

```bash
# Анализ проекта и просмотр в терминале
locus analyze

# Экспорт в Markdown-отчёт
locus analyze -o report.md

# Экспорт в виде модульных текстовых файлов (для LLM-чата)
locus analyze -o output_dir

# Фокус на конкретной директории
locus analyze src/ -o context.md
```

## 📋 Основные команды

### 1. Быстрый обзор

```bash
# Интерактивный просмотр с краткими описаниями
locus analyze -p

# Плоский список файлов (удобен для grep)
locus analyze -p -f
```

### 2. Экспорт для LLM-чата

```bash
# Экспорт с модульной группировкой (рекомендуется)
locus analyze -o llm_context

# Создаст организованные файлы, такие как:
# - src_myapp_features.txt  (все файлы features/* в одном файле)
# - src_myapp_main.py.txt   (main.py отдельно)
# - conf.txt                (все конфигурационные файлы вместе)
```

### 3. Создание конфигурации

```bash
# Инициализация проекта с конфигурацией по умолчанию
locus init --config

# Отредактируйте .locus/settings.json для настройки правил группировки
```

### 4. Целевой экспорт

```bash
# Включить только Python-файлы
locus analyze -o context --include "**/*.py"

# Исключить тесты и миграции
locus analyze -o context --exclude "tests/**" "**/migrations/**"

# Ограничить глубину импортов
locus analyze src/main.py -d 2 -o context.md
```

## 🎨 Режимы вывода

| Режим | Команда | Применение |
|-------|---------|------------|
| **Интерактивный** | `locus analyze` | Быстрый просмотр в терминале |
| **Markdown-отчёт** | `locus analyze -o report.md` | Документация в одном файле |
| **Модульный экспорт** | `locus analyze -o output_dir` | Контекст для LLM-чата (несколько организованных файлов) |

## 🔧 Конфигурация модульного экспорта

Создайте `.locus/settings.json` для настройки группировки файлов:

```json
{
  "modular_export": {
    "enabled": true,
    "max_lines_per_file": 5000,
    "grouping_rules": [
      {
        "pattern": "src/*/features/*",
        "group_by": "module",
        "depth": 3
      },
      {
        "pattern": "src/*/main.py",
        "group_by": "file",
        "separate": true
      }
    ],
    "default_depth": 2
  }
}
```

**Стратегии группировки:**
- `module` - Группировка по пути модуля (например, `src_myapp_features.txt`)
- `directory` - Группировка по директории на указанной глубине
- `file` - Каждый файл отдельно, если `separate: true`

## 📖 Пример рабочего процесса с LLM

### 1. Подготовка контекста для новой фичи

```bash
# Экспорт только релевантных модулей
locus analyze src/features src/utils -o llm_context --include "**/*.py"
```

### 2. Копирование файлов в чат

```bash
# Файлы уже сгруппированы по модулям
ls llm_context/
# src_features.txt
# src_utils.txt

# Скопируйте содержимое в ваш LLM-чат:
cat llm_context/src_features.txt
```

### 3. Обновление кода на основе ответа LLM

```bash
# Используйте locus update для применения изменений из markdown-вывода LLM
locus update < llm_response.md
```

## 🎓 Расширенные возможности

### Извлечение аннотаций

```bash
# Включить docstring'и и сигнатуры
locus analyze -o context -a

# Фокус на интерфейсах
locus analyze -p --no-code
```

### Поиск дубликатов

```bash
# Найти паттерны дублирующегося кода
locus analyze --similarity -o report.md

# Или используйте специальную команду sim
locus sim src/ -s ast -j similarity.json
```

### Обновление файлов из Markdown

```bash
# Предпросмотр изменений
locus update --dry-run < changes.md

# Применить изменения с резервным копированием
locus update --backup < changes.md
```

## 📊 Примеры вывода

### Модульный экспорт (для LLM)

```
output_dir/
├── src_myapp_features.txt      # Все файлы фич сгруппированы
├── src_myapp_utils.txt         # Утилиты сгруппированы
```

```python
def get_profile(user_id):
    """Получить профиль пользователя."""
    # ... реализация
```

## ⚙️ Справка по командам

| Команда | Описание |
|---------|----------|
| `locus analyze [paths]` | Анализ кода и генерация вывода |
| `locus init --config` | Создать конфигурацию по умолчанию |
| `locus update` | Обновить файлы из markdown-ввода |
| `locus sim` | Запустить поиск сходства/дубликатов |

### Ключевые опции

| Опция | Описание |
|-------|----------|
| `-o, --output PATH` | Выходной файл (.md) или директория (модульный экспорт) |
| `-p, --headers` | Включить заголовки и краткие описания файлов |
| `-a, --annotations` | Извлечь docstring'и и сигнатуры |
| `-d, --depth N` | Глубина импортов: 0=выкл, 1=прямые, 2=вложенные, -1=неограниченно |
| `--include PATTERN` | Glob-паттерны для включения (например, `**/*.py`) |
| `--exclude PATTERN` | Glob-паттерны для исключения (например, `tests/**`) |

## 🔗 Сценарии использования

**Для контекста LLM-чата:**
- Экспорт конкретных модулей для предоставления целевого контекста

## 💡 Советы

1. **Начните с малого**: Используйте `locus analyze -p` для первого обзора
2. **Используйте модульный экспорт**: `-o output_dir` лучше, чем один файл для LLM
3. **Настройте группировку**: Кастомизируйте `.locus/settings.json` под структуру вашего проекта
4. **Фильтруйте разумно**: Используйте `--include` и `--exclude` для уменьшения шума
5. **Проверяйте размер вывода**: Большие экспорты могут превышать лимиты контекста LLM
